# NanoTero 🐦

**NanoTero** is an ultra-minimalist, lightweight, and high-performance library designed to deserialize `.tero` configuration files directly into type-safe Rust structures.

Built on a *zero-copy* architecture, a recursive descent approach, and SIMD operations at its lexical core, it provides predictable and efficient syntax parsing without bloating memory.

---

## 🚀 Key Features

* **Minimalist API:** A single entry point to deserialize your data.
* **Type-Safe Deserialization:** Directly maps text into native Rust structs using derive macros.
* **Typed Error Handling:** Absolute control over lexical, syntactic, and evaluation failures.
* **High Performance:** String processing optimized through vector operations (*branchless* SIMD) and exact UTF-8 validation.

---

## 🛠️ Public API (All You Need)

NanoTero keeps its API surface to a minimum to ensure maximum compilation speed and ease of use. The library exports only the essential components:

### 1. `nanotero::from_str`

The main function and engine of the library. It takes a string slice containing the Tero source code and reconstructs any type that implements the `Deserialize` trait.

```rust
#[inline(always)]
pub fn from_str<T: Deserialize>(code: &str) -> Result<T, TeroError>;

```

### 2. `#[derive(TeroDeserialize)]`

The core derive macro. It allows you to mark your structs so they can be automatically generated by the parser. It supports the auxiliary attribute `#[tero(name = "...")]` to rename properties when the configuration file field does not match the Rust identifier exactly.

```rust
#[derive(TeroDeserialize)]
struct Config {
    #[tero(name = "server_ip")]
    ip: String,
    port: u32,
}

```

### 3. `nanotero::TeroError`

The unified enum that manages the entire error flow of the system. It tells you exactly whether the process failed during the lexical, syntactic, or type-mapping (evaluation) phase.

### 4. `nanotero::Deserialize`

The trait governing type conversion. It is natively implemented for primitives (`i64`, `f64`, `bool`, `String`), wrappers (`Box`, `Cow`, `Option`), and standard collections (`BTreeMap`).

## 🎨 Usage Example

Here is a complete example of how to integrate **NanoTero** into your Rust project using the new strongly-typed API:

```rust
use nanotero::{from_str, TeroDeserialize, TeroError};

// Define the expected structure of the configuration
#[derive(TeroDeserialize, Debug)]
struct ServerConfig {
    // Map "server_name" in the .tero file to the "name" field
    #[tero(name = "server_name")]
    name: String,
    port: u32,
    active: bool,
    modes: Vec<String>,
}

fn main() {
    // Your text string in Tero format
    let config_code = r#"
        server_name: "KiwiServer"
        port: 8080
        active: true
        modes [ "production", "debug" ] # Integrated comment
    "#;

    // Direct deserialization to the target struct using type inference
    let result = from_str::<ServerConfig>(config_code).unwrap();
}
```
